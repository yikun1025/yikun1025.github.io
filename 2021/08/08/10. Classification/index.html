10.Classification

发表于2021-08-08|更新于2022-06-20|深度学习笔记
|阅读量:

Classification Example

关于分类问题，假设输入是金字图片，那么分类就是这个字属于哪个字。

输入是关于个人金融相关信息，输出就是同意贷款or拒绝贷款

假设输入是医疗诊断，目前的症状，输出就是分类属于哪种疾病

Example Application

对于宝可梦的问题，假设我们的input输入是皮卡丘，分类是电，input杰尼龟，输出是水，input是妙蛙种子，输出是草

但是我们如何将pokemon，也就是将这个input数值化呢？pokemon会有七个数值，这七个数值可以组成一个数组。

total就是hp,attack, defense,sp atk, sp df,speed总和的值，将七个数字分入到一个function。

how to do classification

将pokemon 编号400以下的数据当训练集，400以上编号的数据当测试集。假设我们分类为二分类问题，取值范围为1,-1，那么得出来结果偏1的就是class1, 结果偏-1的就是class2

假设我们用回归的问题做分类问题，就可能会在训练的时候遇到这样的问题

可以看到右图中出现了一些error值，这些error值远远大于1，因此会导致线右偏。我们理想的是绿色的线，也就是人类肉眼所寻找的线，但是实际上因为蓝色erorr的位置，最终会找到的是紫色这条线。

因此我们应该用其他方式来解决

ideal Alternatives

假设为二元分类问题。我们可以找一个function，让output的结果带入这个g function，如果g(x)>0那么就是class1,否则就是class2。

那么loss function也可以修改，这个就可以改为在training data上预测的分类错误次数，如果错误+1，因此loss也是越小越好。

two boxes

假设盒子1抽一颗球的概率为2/3，盒子2抽一颗球概率为1/3，蓝球在盒子1的概率为4/5, 在盒子2概率为2/5

那么如果抽到蓝色球，并且是从盒子1的概率为多少。这里其实有用到贝叶斯的方法

那么把蓝色改为class，其实公式也一样

可见，问题就可以归纳为分类问题的解决方法。那么我们需要知道的参数就转化为，需要知道从class1抽一个x的概率，从class2抽一个x概率，抽到class1,class2的概率。从class1里抽到x的概率，从class2中抽到x的概率。那么就可以求出x属于class 1的几率。

从c1挑出x的几率乘以从所有class中挑出c1的几率，加c2挑出x的几率乘以从所有class中挑出c2的几率，那么就是可以得到的对于x的概率。

prior

先验为从class中挑选出的概率。首先我们可以选择种类数最多的做二分类，目前pokemon有18分类，因此这个问题会很复杂，从二分类开始预测，我们将前400只pokemon选种类最多的两个类做分类，分别为79只水系和61只一般系。

那么计算这两个分类的概率

probability from class

我们现在知道有一只海龟，那么如何预测它是否是水系呢

probability from class - feature

假设我们只考虑两个特征，那就是防御力和特殊防御力，并把这些pokemon都作图，可以看到如图所示

我们画图的是training的这些pokemon，但是我们要找的海龟并不在这里面，那么怎么求概率呢，不能是0。

Gaussian distribution

这里就可以用到高斯分布，从统计可以知道分布大部分都是基于高斯分布的。那么可以从我们已知的training的pokemon就可以画出概率分布图。

不同的数据，获得的高斯分布的成都不同，通过这个复杂式子就可以得到海龟在这个水系高斯分布中存在的概率为多少。

function越接近红色也就是$\mu$​，概率越大。

maximum likelihood

这里的方式就是我们算不同的$\mu,\sum$，找到最大可能分布这79个input的高斯分布，也就是覆盖的点越多就是我们要找的方程，这里运用到了最大似然概率方法的思想。

这个思想就是假设我们观测到的分类数量越多，真实这个分类概率就是越多的。比如我们在同时间中洗手间观测到女生比较多，那么我们可以合理推测同时间中去洗手间的女生是比去洗手间的男生多的。一种非常朴素的统计方法。

result

对于water和normal做出来的结果是这样的。

如果值大于0.5就说明是属于类别1，也就是水类。

做了二分类后会发现，这个效果很差，只有百分之47的准确率，红色区域是对于水系的类别集合，蓝色区域是对于一般系的集合。

resut adding more traits

假设我们增加更多维度，是不是会分布更好呢，会不会处理的效果更好，比如说我们的特征增加为七个，也就是包括hp，attack，sp attack, de, sp de,speed，然而正确率仍然很低，只有百分之54的精度。

modifying model

这里模型提升是基于water和normal用的covariance matrix为同一个，只是$\mu$不同，这样还是用乘积结合为likelihood
